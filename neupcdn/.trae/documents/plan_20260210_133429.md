I will transform the current `neupcdn` Go application into the **CDN App (Storage + Verifier)** as specified. This involves stripping out the direct-upload logic and implementing a strict verification flow that trusts only `neup.drive`.

### 1. Configuration & Security Refactor
- **Remove**: `AccountPublicKeys` and per-account Ed25519 logic.
- **Add**: `SHARED_SECRET` (for verifying HMAC tokens from `neup.drive`).
- **Add**: `NEUP_DRIVE_SSH_KEY` (to strictly authorize the Node.js app via SFTP).
- **Define**: `TEMP_DIR` (staging) vs `PUBLIC_ROOT` (final storage).

### 2. SFTP Server Overhaul (The "Staging Area")
- **Restrict Access**: Only accept connections from the `neup.drive` user with the authorized SSH key.
- **Chroot**: Lock the SFTP session to `TEMP_DIR`.
- **Behavior**: Files uploaded here are temporary. The filename will be the `upload_session_id`.
- **Remove**: The custom "Jail" logic that separated files by account/category. The CDN is now "dumb" storage; organization is handled by `neup.drive` via the final move command.

### 3. New HTTP API: `POST /verify-upload`
This is the core logic. It will:
1.  **Receive Payload**: `session_id`, `computed_hash`, `size`, `mime`, and the `signed_token`.
2.  **Verify Token**: Check the HMAC signature of the token using `SHARED_SECRET` to ensure it was issued by `neup.drive`.
3.  **Validate Integrity**:
    *   Check if the file exists in `TEMP_DIR` (named via `session_id`).
    *   Compute the SHA256 hash of the file on disk.
    *   Compare it with the `computed_hash` provided by `neup.drive`.
    *   Verify file size matches the token's `expected_size`.
4.  **Finalize**:
    *   If all checks pass: **Move** the file to `PUBLIC_ROOT` + `token.destination_path`.
    *   If any check fails: **Delete** the temp file immediately.

### 4. Cleanup
- Remove `POST /upload` and `POST /upload/init` (direct client uploads are forbidden).
- Clean up unused code related to the old security model.
